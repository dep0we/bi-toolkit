# Glossary

Plain definitions for terms used in this guide and the toolkit.

Previous: [Troubleshooting FAQ](11-troubleshooting-faq.md) | [Index](README.md)

## A

**A/B test**: comparison between two versions.

**Absolute tolerance**: allowed raw-number movement.

**Acceptance criteria**: conditions required before work is done.

**Active analysis**: saved work to resume first.

**Adversarial review**: independent attack on weak spots.

**Aggregate**: grouped summary, not individual rows.

**AI**: software that drafts and checks work.

**Alias**: another command name for same action.

**Analysis**: one-time answer using data.

**Analysis id**: short saved-work name.

**Anomaly**: unexpected value needing review.

**ARR**: annual recurring revenue.

**Assay loop**: staged path to trusted analysis.

**Assumption**: treated as true without full proof.

**Audit trail**: saved evidence of decisions and checks.

## B

**Baseline**: normal comparison point.

**BI**: business reporting and metrics.

**Block**: required stop before continuing.

**Business logic**: rules that define business meaning.

## C

**Caveat**: limit that affects trust.

**Chart library**: code used to draw charts.

**Claude Code**: AI assistant run in a folder.

**Cohort**: group tracked over time.

**Command**: text the computer runs.

**Confidence**: how sure the answer is right.

**Confidence interval**: likely range around estimate.

**Confirmation bias**: seeing only supporting evidence.

**Contract**: required shape for input data.

**Conversion**: people reaching the next step.

**Correlation**: two measures moving together.

**Cross-family review**: different model family checks work.

**CSV**: plain spreadsheet-style text file.

## D

**Dashboard**: recurring view of business metrics.

**Data classification**: how sensitive the data is.

**Data completeness**: how much needed data exists.

**Data product**: recurring report or dashboard.

**Data quality**: fitness of data for use.

**Decision impact**: business choice the answer affects.

**Decision ledger**: saved list of ruled choices.

**Deliverable**: file or answer given to people.

**Distribution**: who receives report and where.

**Distribution manifest**: delivery handoff record.

**Drift**: unexpected metric movement.

**Driftcheck**: recurring metric movement check.

## E

**EDA**: first-pass data inspection.

**Eligibility rule**: who or what is included.

**Evidence**: proof supporting a conclusion.

**Execute**: run ruled analysis or build output.

**Export**: data leaves analysis workspace.

## F

**Fail-closed**: block until proof exists.

**Filter**: rule excluding some records.

**Forecast**: future estimate with uncertainty.

**Fork**: choice that changes numbers.

**Frame**: name decision and work track.

**Fresh agent**: independent worker not previously involved.

## G

**Gate**: required stop-check before continuing.

**Git**: file version tracking tool.

**Governing docs**: protected project rule files.

**Grain**: what one row represents.

**Guardrail metric**: must-not-get-worse measure.

## H

**High-stakes**: drives money, headcount, or strategy.

**HTML**: browser page file format.

## I

**Impact quantification**: sizing business effect.

**In-flight analysis**: saved unfinished analysis work.

**Inline definition**: short explanation beside term.

**Intake**: capture project tools and rules.

**Internal data**: company-only business data.

**IQR**: middle-half spread.

## J

**Join**: matching records across tables.

**JSON**: structured key-value data.

**JSONC**: JSON with comments.

## K

**KPI**: main number watched for decisions.

## L

**Ledger**: saved history of decisions.

**Line chart**: chart showing values over time.

**Lineage**: where data comes from.

**Living metric store**: maintained metric definitions.

## M

**MAD**: typical distance from middle.

**Main model**: AI leading judgment.

**MAPE**: average percent miss.

**Mean**: average value.

**Median**: middle value, ignores outliers.

**Metric**: number used for decisions.

**Metric catalog**: shared metric definition file.

**Metric definition**: exact calculation rule.

**Metric store**: maintained metric definition workflow.

**Methodology**: chosen analysis approach.

**Methodology fork**: choice that changes numbers.

**Methodology ruling**: approved method choice.

**Model**: AI system doing thinking.

**Model dial**: cost-and-quality model choice.

## N

**Non-zero**: command ended with failure.

**Null**: blank or unknown value.

## O

**Operator**: person guiding the analysis.

**Outlier**: unusual value that can skew.

## P

**p-value**: chance result is noise.

**Panel**: dashboard section.

**PATH**: command lookup list.

**PDF**: print-ready file for sharing.

**PHI**: health information.

**PII**: personal identifying information.

**Plain language**: wording a business reader understands.

**Power**: chance to detect real effect.

**Preflight**: gate check before chokepoint.

**Print-to-PDF**: browser saves page as PDF.

**Project folder**: folder for one work area.

**Proxy metric**: stand-in measure.

## Q

**Query**: request for data.

**Query language**: how data is requested.

## R

**Receipt**: saved proof file.

**Reconcile**: match or explain differences.

**Reconciliation**: source comparison with explained variance.

**Refresh**: new data arrives.

**Relative tolerance**: allowed percent movement.

**Refresh cadence**: how often data updates.

**ReproCommand**: command rerunning the work.

**Reprocheck**: reproducibility delivery gate.

**Reproducibility**: rerun gets same answer.

**Residual**: leftover movement after patterns.

**Retention**: customers or users staying.

**ROI**: return compared with investment.

**Row-level detail**: individual records are shown.

**Ruling**: operator-approved method choice.

## S

**Sample size**: number of records or units.

**Schema**: map of tables and columns.

**Score threshold**: minimum allowed review score.

**Seasonality**: repeating calendar pattern.

**Segment**: group with shared traits.

**Semantic layer**: shared metric definitions.

**Sensitive data**: personal, health, payroll, customer data.

**Shell script**: small command program.

**Significance level**: accepted false-win chance.

**Source-of-truth**: official place to compare.

**Spec**: written question and metric rules.

**Spec receipt**: proof the spec exists.

**SQL**: language for database queries.

**Stakeholder**: person affected by work.

**Static HTML**: browser page saved as file.

**Sub-agent**: worker agent with narrow task.

## T

**Tableau**: BI dashboard tool.

**Technical term**: specialized word needing definition.

**Terminal**: text command window.

**Threshold**: minimum or maximum allowed value.

**Time series**: values measured in time order.

**Track**: analysis or data-product path.

**Trend**: longer-term direction.

**Tripwire**: rule that forces escalation.

## U

**Uncertainty**: amount we may be wrong.

## V

**Valid answer**: complete answer by agreed rule.

**Validation**: checking result before delivery.

**Validation receipt**: proof checks were done.

**Variance**: difference from expected value.

**Versioning**: keeping named output versions.

## W

**Warehouse**: central store for analysis data.

**Workflow**: ordered steps for work.

## Z

**Zero exit**: command ended successfully.
